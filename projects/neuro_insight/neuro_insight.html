Neuro Insight - Final year project

Project Overview

NeuroInsight is a medical web application designed to assist doctors in diagnosing glioma brain tumors from MRI scans using deep learning.

It automates:

  • Tumor detection (presence/absence)
  • Multi-region segmentation (edema, necrotic core, enhancing tumor)
  • Grade classification (High-Grade vs. Low-Grade Glioma)
  • PDF report generation for clinical use

Built with Django (Python) and state-of-the-art AI models, it bridges the gap between research and clinical practice by offering a non-technical user interface for medical professionals.

Project Highlights
  • Classification Accuracy 98.7%
  • Segmentation Dice Score 0.99
  • Diagnosis Time Reduction Hours → Seconds

Technical Implementation

Data Splitting Strategy

To train and evaluate our model effectively we split our dataset into three parts:

Initial Split
  • 70% of the data for training
  • 30% of the data reserved for testing and validation
Second Split
  • 15% for testing (from the 30% reserved)
  • 15% for validation (from the 30% reserved)

Stratified Splitting: Used to preserve the distribution of High-Grade Glioma (HGG) and Low-Grade Glioma (LGG) samples in each set and avoid class imbalance issues.

Data Preprocessing Pipeline

High resolution 3D MRI scans consume significant memory, so we implemented a data generator with preprocessing steps for memory efficiency:

Z-score Normalization

Transforms data to have a mean of 0 and standard deviation of 1, important for medical imaging where pixel values can range widely (e.g., max value 1273.0).

Formula: z = (x - μ) / σ
Slice Selection

The generator uses slices restricted by a range (default 50 to 130) to reduce memory use and focus on diagnostically relevant slices.

One Hot Encoding

Segmentation mask is resized to 128x128 pixels, and one-hot encoding is applied to create a categorical mask with 4 classes.

Segmentation Mask Processing

The '__load_segmentation' function loads the segmentation mask and converts all class 4 labels into class 3 to simplify model training.

Model Architecture: Attention U-Net for Tumor Segmentation

Our project employs a modified U-Net architecture integrated with attention gates, inspired from the paper "Attention U-Net: Learning Where to Look for the Pancreas" (Oktay, 2018).

Component Details
Input 128 × 128 × 2 (FLAIR and T1CE MRI slices)
Encoder 3 blocks of Conv2D (ReLU) + MaxPooling with filters: 32 → 64 → 128
Bottleneck 2 Conv2D layers with 256 filters each
Attention Gates Applied before each decoder skip connection (128, 64, 32 filters)
Decoder 3 UpSampling blocks with Conv2D layers and concatenated attention features
Output Layer 1 × 1 Conv2D + Softmax (for 4-class segmentation)
Loss Function Categorical Crossentropy
Optimizer Adam (learning rate = 0.001)
Batch Size 2 (used for memory efficiency)
Precision Mode Mixed Precision Training (float16)
